1. ¿Qué es y para qué sirve?1. What is it and what is it for?

El conector Google Chat permite que el agente de Ruvic envíe mensajes a los espacios (salas) de Google Chat de tu organización. Una vez configurado, puedes pedirle al agente cosas como:

  • “Cuando termine el despliegue, avisa en el espacio de Ingeniería de Google Chat.”
  • “Publica el resumen de ventas de hoy en el espacio de Dirección.”
  • “Si una tarea de facturación falla, notifícalo en el espacio de Operaciones.”

Operaciones soportadas:

The Google Chat connector lets the Ruvic agent send messages to your organization's Google Chat spaces (rooms). Once configured, you can ask the agent things like:

  • “When the deploy finishes, post in the Engineering space on Google Chat.”
  • “Publish today's sales summary in the Leadership space.”
  • “If a billing job fails, notify the Operations space.”

Supported operations:

OperaciónOperation Qué haceWhat it does
chat.send_message Envía un mensaje de texto a un espacio (soporta hilos)Sends a text message to a space (supports threads)
chat.list_spaces Lista los espacios donde la app está agregada (solo modo cuenta de servicio)Lists the spaces where the app is added (service-account mode only)
chat.get_space Consulta la información de un espacio (solo modo cuenta de servicio)Gets information about a space (service-account mode only)
Dos modos de conexiónTwo connection modes El conector ofrece dos formas de conectarse. El webhook es el camino rápido: sin proyecto de Google Cloud, pero solo envía a un espacio. La cuenta de servicio (Chat API) requiere configurar Google Cloud, pero permite enviar a cualquier espacio donde agregues la app, listar espacios y consultar su información. Elige uno según tus necesidades. The connector offers two ways to connect. The webhook is the fast path: no Google Cloud project, but it sends to a single space only. The service account (Chat API) requires Google Cloud setup, but lets you send to any space where you add the app, list spaces and query their info. Pick one based on your needs.

2. Prerrequisitos2. Prerequisites

Comunes a ambos modos:

  • Una cuenta de Google Workspace con acceso a Google Chat y al menos un espacio destino.
  • Permiso para gestionar apps/webhooks en ese espacio (normalmente, ser miembro con derechos de administración del espacio).

Solo para el modo cuenta de servicio:

  • Acceso a Google Cloud Console con permiso para crear proyectos, habilitar APIs y crear cuentas de servicio.
  • En organizaciones restringidas, que el administrador de Workspace permita apps de Chat de terceros/internas.

Common to both modes:

  • A Google Workspace account with access to Google Chat and at least one target space.
  • Permission to manage apps/webhooks in that space (usually being a member with space management rights).

Service-account mode only:

  • Access to Google Cloud Console with permission to create projects, enable APIs and create service accounts.
  • In restricted organizations, the Workspace admin must allow third-party/internal Chat apps.

3. Configuración en Google (paso a paso)3. Google-side setup (step by step)

Elige tu rutaChoose your path Sigue solo la Opción A (webhook) o solo la Opción B (cuenta de servicio), según el modo que vayas a usar. No necesitas ambas. Follow only Option A (webhook) or only Option B (service account), depending on the mode you'll use. You don't need both.

Opción A — Webhook entrante (rápido, un espacio)Option A — Incoming webhook (fast, single space)

  1. Abre el espacio destino en Google Chat.

    En chat.google.com (o Gmail con Chat activado), entra al espacio donde quieres que el agente publique. Los webhooks solo existen en espacios, no en mensajes directos.

    Open the target space in Google Chat.

    In chat.google.com (or Gmail with Chat enabled), open the space where you want the agent to post. Webhooks only exist in spaces, not in direct messages.

  2. Añade un webhook.

    Pulsa el nombre del espacio → Apps e integraciones → Webhooks → Añadir webhook. Asigna un nombre (ej. “Ruvic”) y, opcionalmente, un avatar.

    Add a webhook.

    Click the space name → Apps & integrations → Webhooks → Add webhook. Give it a name (e.g. “Ruvic”) and optionally an avatar.

    [ Captura: diálogo “Añadir webhook” en un espacio de Google Chat — reemplazar por assets/img/gchat-webhook.png ] [ Screenshot: “Add webhook” dialog in a Google Chat space — replace with assets/img/gchat-webhook.png ]
  3. Copia la URL del webhook.

    Google genera una URL que incluye los parámetros key y token. Cópiala completa: es la credencial que irá en el formulario de Ruvic (campo URL del webhook). Trátala como un secreto.

    Copy the webhook URL.

    Google generates a URL that includes the key and token parameters. Copy it in full: it is the credential that goes in the Ruvic form (Webhook URL field). Treat it as a secret.

    https://chat.googleapis.com/v1/spaces/AAAA/messages?key=...&token=...

Opción B — Cuenta de servicio (Chat API)Option B — Service account (Chat API)

  1. Crea o elige un proyecto en Google Cloud.

    Entra a Google Cloud Console y selecciona (o crea) el proyecto donde vivirá la app de Chat.

    Create or pick a project in Google Cloud.

    Go to Google Cloud Console and select (or create) the project where the Chat app will live.

  2. Habilita la Google Chat API.

    APIs y servicios → Biblioteca → busca Google Chat API → Habilitar.

    Enable the Google Chat API.

    APIs & Services → Library → search Google Chat API → Enable.

    [ Captura: página de la Google Chat API con el botón Habilitar — reemplazar por assets/img/gchat-enable-api.png ] [ Screenshot: Google Chat API page with the Enable button — replace with assets/img/gchat-enable-api.png ]
  3. Crea una cuenta de servicio y su clave JSON.

    IAM y administración → Cuentas de servicio → Crear cuenta de servicio. Luego, en la cuenta creada: Claves → Agregar clave → Crear clave nueva → JSON. Se descarga un archivo .json: su contenido completo irá en el formulario de Ruvic. Guárdalo de forma segura.

    Create a service account and its JSON key.

    IAM & Admin → Service Accounts → Create service account. Then, on the created account: Keys → Add key → Create new key → JSON. A .json file downloads: its full contents go into the Ruvic form. Store it securely.

    [ Captura: creación de la clave JSON de la cuenta de servicio — reemplazar por assets/img/gchat-sa-key.png ] [ Screenshot: creating the service account JSON key — replace with assets/img/gchat-sa-key.png ]
  4. Configura la app de Chat.

    APIs y servicios → Google Chat API → Configuración: completa el nombre de la app, avatar y descripción, y habilita que pueda usarse en espacios. Sin este paso, la API responde error de permisos.

    Configure the Chat app.

    APIs & Services → Google Chat API → Configuration: fill in the app name, avatar and description, and enable it for spaces. Without this step, the API returns a permissions error.

  5. Agrega la app a cada espacio destino.

    En Google Chat, entra al espacio → ＋ Añadir personas y apps → busca el nombre de tu app de Chat y agrégala. Repite en cada espacio donde el agente deba publicar.

    Add the app to each target space.

    In Google Chat, open the space → ＋ Add people & apps → search your Chat app's name and add it. Repeat for every space where the agent should post.

  6. Anota el ID del espacio por defecto (opcional).

    Abre el espacio en el navegador; el ID aparece en la URL con el formato spaces/AAAAAAAAAAA. Lo usarás en el campo Espacio por defecto (o el agente lo descubrirá con list_spaces).

    Note the default space ID (optional).

    Open the space in the browser; the ID appears in the URL as spaces/AAAAAAAAAAA. You'll use it in the Default space field (or the agent can discover it with list_spaces).

Verifica el flujo actualVerify the current flow Los menús de Google Cloud y Google Chat cambian con frecuencia. Confirma cada paso contra la consola real al momento de configurar; los nombres exactos de los botones pueden variar. Google Cloud and Google Chat menus change often. Confirm each step against the live console when configuring; exact button names may vary.

4. Permisos requeridos4. Required permissions

Principio de mínimo privilegio: el conector solo necesita publicar mensajes como la app y, en modo cuenta de servicio, listar/consultar los espacios donde está agregada.

Least-privilege principle: the connector only needs to post messages as the app and, in service-account mode, list/read the spaces where it is added.

Permiso / scopePermission / scope ModoMode Para qué lo usa el conectorWhy the connector needs it
Webhook activo en el espacioActive webhook in the space Webhook Publicar mensajes en ese espacioPost messages to that space
https://www.googleapis.com/auth/chat.bot Cuenta de servicioService account Enviar mensajes como la app, listar espacios y consultar su informaciónSend messages as the app, list spaces and read their info
App agregada al espacioApp added to the space Cuenta de servicioService account Sin ser miembro, la API devuelve “no encontrado” al intentar publicarWithout membership, the API returns “not found” when trying to post

El scope chat.bot es de autenticación de app: el agente actúa como la app de Chat, no en nombre de un usuario, por lo que no requiere delegación de dominio para enviar mensajes.

The chat.bot scope is app authentication: the agent acts as the Chat app, not on behalf of a user, so it doesn't require domain-wide delegation to send messages.

6. Verificación6. Verification

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa…” (indicando el modo webhook o cuenta de servicio). La tarjeta del conector cambia a estado Configurado. Este test no publica mensajes en el espacio.

    Successful connection test.

    The Test connection button should return: “Successful connection…” (indicating webhook or service-account mode). The connector card switches to Configured. This test posts no messages to the space.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele:

    “Envía un mensaje a Google Chat que diga: Conector configurado ✅.”

    El mensaje debe aparecer en el espacio destino. Si llega, el conector quedó operativo de punta a punta.

    Minimal test automation.

    Open a new conversation with the agent and ask:

    “Send a message to Google Chat saying: Connector configured ✅.”

    The message should appear in the target space. If it arrives, the connector is fully operational.

8. Preguntas frecuentes y notas de seguridad8. FAQ and security notes

¿Qué modo debería usar?Which mode should I use?

Si solo necesitas publicar en un espacio y quieres lo más simple, usa el webhook. Si necesitas enviar a varios espacios, listar espacios o consultarlos, usa la cuenta de servicio.

If you only need to post to one space and want the simplest setup, use the webhook. If you need to send to several spaces, list them or query them, use the service account.

¿El agente puede leer los mensajes del espacio?Can the agent read the space's messages?

No. Este conector es de solo envío (más listar/consultar espacios en modo cuenta de servicio). No lee el historial de mensajes de los espacios.

No. This connector is send-only (plus listing/reading spaces in service-account mode). It does not read the spaces' message history.

¿Dónde quedan guardadas las credenciales?Where are the credentials stored?

Tanto la URL del webhook como el JSON de la cuenta de servicio se guardan como secretos en el almacén de credenciales de la plataforma. La interfaz nunca vuelve a mostrarlos; solo indica “definido”.

Both the webhook URL and the service account JSON are stored as secrets in the platform's credential store. The UI never displays them again; it only shows “set”.

Rotación de credencialesCredential rotation

Webhook: elimina el webhook en el espacio y crea uno nuevo; actualiza la URL en el formulario. Cuenta de servicio: en Google Cloud, crea una nueva clave JSON, actualízala en el formulario y luego elimina la clave anterior.

Webhook: delete the webhook in the space and create a new one; update the URL in the form. Service account: in Google Cloud, create a new JSON key, update it in the form, then delete the old key.

¿Qué hago si una credencial se compromete?What if a credential is compromised?

  1. Webhook: elimínalo de inmediato en el espacio (invalida la URL) y crea uno nuevo.
  2. Cuenta de servicio: en Google Cloud, elimina/deshabilita la clave JSON comprometida (IAM → Cuentas de servicio → Claves).
  3. Actualiza la nueva credencial en el formulario del conector.
  1. Webhook: delete it immediately in the space (invalidates the URL) and create a new one.
  2. Service account: in Google Cloud, delete/disable the compromised JSON key (IAM → Service Accounts → Keys).
  3. Update the new credential in the connector form.
Buenas prácticasBest practices Usa una cuenta de servicio dedicada solo para Ruvic; no compartas el JSON ni la URL del webhook por canales no seguros ni los publiques en repositorios; agrega la app únicamente a los espacios necesarios; y rota las claves periódicamente según tu política. Use a dedicated service account only for Ruvic; never share the JSON or the webhook URL over insecure channels or commit them to repositories; add the app only to the spaces needed; and rotate keys periodically per your policy.
