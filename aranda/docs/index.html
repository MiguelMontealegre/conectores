Conector Aranda ASMS

Última actualización: 21 de julio de 2026

1. ¿Qué es y para qué sirve?

El conector Aranda ASMS permite que el agente de Ruvic gestione casos en el service desk de Aranda: abrir incidentes, requerimientos, problemas o cambios, consultar casos y resolver toda la información necesaria (proyectos, servicios, categorías, estados, clientes…). Una vez configurado, puedes pedirle al agente cosas como:

  • “Abre un incidente en Aranda: el servidor de correo no responde.”
  • “Consúltame el estado del caso IM-10658.”
  • “Lístame los servicios del proyecto de Soporte para incidentes.”

Grupos de operaciones soportadas:

Grupo Qué hace
Casos Crear un caso (create_item) y consultarlo por su id (get_item_by_id)
Catálogos Tipos de caso, urgencia, impacto y tipos de registro
Descubrimiento Proyectos, servicios, categorías, modelos, estados, compañías, clientes y campos adicionales para armar el caso
Flujo y asignación Transiciones de estado, grupos, motivos y responsables
CMDB Búsqueda de elementos de configuración (CIs)
Actúa con un token de integración Ruvic opera Aranda mediante un token de integración que tú generas en la consola de Aranda. Todas las acciones se ejecutan con los permisos asociados a ese token.

2. Prerrequisitos

  • Una instalación de Aranda ASMS con la API v9 habilitada (nube de Arandasoft u on-premise).
  • Un token de integración con permisos sobre los proyectos y tipos de caso que se vayan a gestionar.
  • La URL base de la API de tu instalación (para on-premise). En la nube de Arandasoft es la que trae el conector por defecto.

3. Configuración en Aranda (paso a paso)

Vas a obtener un token de integración de Aranda ASMS y a confirmar la URL base de la API. El token es la única credencial secreta.

  1. Genera el token de integración.

    En la consola de administración de Aranda ASMS, entra a la sección de integraciones / API y genera (o copia) el token de integración para el usuario o la aplicación que operará los casos. Guárdalo: es la credencial que irá en el formulario de Ruvic.

    Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...
    [ Captura: pantalla de token de integración en la consola de Aranda (token ofuscado) — reemplazar por assets/img/aranda-token.png ]
  2. Confirma la URL base de la API.

    Es la ruta de la API v9 de tu instalación. En la nube de Arandasoft es https://proyectos.arandasoft.com/asmsapi/api/v9. En on-premise, la URL de tu servidor terminando en /asmsapi/api/v9.

  3. Verifica los permisos del token.

    El token debe poder ver y operar los proyectos y tipos de caso que Ruvic vaya a gestionar (crear casos, consultar, listar catálogos). Si falta algún permiso, pídeselo al administrador de Aranda.

4. Permisos requeridos

Principio de mínimo privilegio: usa un token con acceso solo a los proyectos necesarios. El conector no hace más de lo que el token permite.

Acceso del token Para qué lo usa el conector
Lectura de catálogos y proyectos Listar tipos de caso, urgencia, impacto, proyectos, servicios, categorías, estados, etc.
Creación de casos Abrir incidentes/requerimientos/problemas/cambios (create_item)
Consulta de casos Leer el detalle de un caso por su id (get_item_by_id)
Acceso a la CMDB Buscar elementos de configuración (search_cis)

5. Configuración en Ruvic

Entra a Settings → Conectores, busca la tarjeta Aranda ASMS y pulsa Configurar. Diligencia el formulario con los valores de la sección 3:

Campo del formulario Valor (obtenido en la sección 3)
Token de integración El token del paso 1 (con o sin Bearer). Se guarda cifrado como secreto: la interfaz no volverá a mostrarlo
URL base de la API Opcional. La de tu instalación; si se omite, se usa la nube de Arandasoft
Validar certificado TLS Opcional. Déjalo desactivado si tu servidor usa certificado interno (por defecto)
Timeout de peticiones Opcional; 30 segundos por defecto

Guarda y pulsa Probar conexión — la plataforma validará el token listando los tipos de caso disponibles.

6. Verificación

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa con Aranda ASMS; N tipos de caso disponibles.”. La tarjeta cambia a estado Configurado.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele:

    “Lístame los proyectos que tengo disponibles en Aranda.”

    Si el agente devuelve la lista de proyectos, el conector quedó operativo. Crear un caso requiere resolver primero varios IDs (servicio, categoría, modelo, estado, cliente); el agente lo hace paso a paso.

7. Solución de problemas

Mensaje / síntoma Causa probable y solución
“Autenticación o permisos insuficientes (HTTP 401/403)” Token inválido, expirado o sin permisos. Genera uno nuevo en la consola de Aranda y actualízalo en el formulario.
Error de certificado TLS Tu servidor usa un certificado interno. Deja Validar certificado TLS desactivado (comportamiento por defecto).
“No se pudo alcanzar Aranda” URL base incorrecta o servidor inaccesible. Verifica la URL (debe terminar en /asmsapi/api/v9) y la conectividad de red.
“Recurso no encontrado” / “rechazó la operación” Algún ID (proyecto, servicio, categoría, modelo, estado, cliente) no existe o la combinación no es válida. Resuélvelos con las operaciones de descubrimiento antes de crear el caso.
La creación falla por campos obligatorios Faltan campos adicionales obligatorios. Consulta los campos requeridos con la operación de campos adicionales y complétalos.

8. Preguntas frecuentes y notas de seguridad

¿Con qué usuario se crean los casos?

Con la identidad/permisos asociados al token de integración. Los casos y acciones se registran conforme a ese token.

¿Dónde queda guardado el token?

Como secreto en el almacén de credenciales de la plataforma, asociado a tu usuario. La interfaz nunca vuelve a mostrarlo; solo indica “definido”. Para cambiarlo, escribe uno nuevo; para conservarlo al editar otros campos, deja el campo vacío.

¿Por qué la validación TLS viene desactivada?

Muchas instalaciones on-premise de Aranda usan certificados internos que no son de confianza pública. Por eso el conector no valida TLS por defecto. Si tu servidor tiene un certificado válido y de confianza, puedes activar la validación.

¿Qué hago si el token se compromete?

  1. En la consola de Aranda, revoca/regenera el token de integración de inmediato.
  2. Actualiza el nuevo token en el formulario del conector.
  3. Revisa en Aranda los casos y acciones recientes por actividad no autorizada.
Buenas prácticas Usa un token dedicado con acceso solo a los proyectos necesarios; no compartas el token por canales inseguros ni lo publiques en repositorios; y rótalo periódicamente según tu política de seguridad.
