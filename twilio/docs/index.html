Conector Twilio SMS

Última actualización: 16 de julio de 2026

1. ¿Qué es y para qué sirve?

El conector Twilio SMS permite que el agente de Ruvic envíe mensajes de texto (SMS) a cualquier número de teléfono y consulte si fueron entregados. Es ideal para mensajes transaccionales. Una vez configurado, puedes pedirle al agente cosas como:

  • “Envía un código de verificación por SMS al número del cliente.”
  • “Cuando el pedido se despache, notifícalo por SMS al comprador.”
  • “¿El SMS de alerta que enviamos hace un momento llegó?”

Operaciones soportadas:

Operación Qué hace
sms.send Envía un SMS a un número en formato internacional
sms.get_status Consulta el estado de entrega de un mensaje enviado
sms.list Lista los mensajes recientes de la cuenta (con filtros)
Envío, no recepción Este conector es de solo salida: envía SMS y consulta su estado. Recibir SMS entrantes requeriría un endpoint público permanente, incompatible con el modelo de ejecución de Ruvic.

2. Prerrequisitos

  • Una cuenta de Twilio (twilio.com/console). Sirve una cuenta trial para pruebas, con limitaciones (ver más abajo).
  • Un remitente: un número de teléfono de Twilio habilitado para SMS, o un Messaging Service.
  • Según el país destino, las verificaciones regulatorias correspondientes (verificación toll-free en EE. UU./Canadá, registro 10DLC, o bundle regulatorio en otros países).
  • Saldo en la cuenta: cada SMS enviado tiene costo (salvo credenciales de prueba).
Cuentas trial Una cuenta trial de Twilio solo puede enviar a números previamente verificados en la consola, y antepone un texto de prueba a cada mensaje. Para producción, actualiza la cuenta.

3. Configuración en Twilio (paso a paso)

  1. Consigue un remitente.

    En la consola de Twilio, ve a Phone Numbers y compra (o usa) un número habilitado para SMS. Anótalo en formato E.164 (ej. +15017122661). Alternativamente, crea un Messaging Service en Messaging → Services y anota su SID (empieza por MG).

    [ Captura: número SMS o Messaging Service en la consola de Twilio — reemplazar por assets/img/twilio-sender.png ]
  2. (Cuenta trial) Verifica el número destino.

    Si tu cuenta es trial, ve a Phone Numbers → Verified Caller IDs y verifica cada número al que quieras enviar SMS de prueba. Sin esto, el envío fallará con el código 21608.

  3. Elige tus credenciales — Opción A: Account SID + Auth Token.

    En el panel principal de la consola verás el Account SID (empieza por AC) y el Auth Token. Cópialos. Es lo más rápido, ideal para pruebas.

    [ Captura: Account SID y Auth Token en el panel (token ofuscado) — reemplazar por assets/img/twilio-credentials.png ]
  4. Opción B (recomendada para producción): crea una API Key.

    Ve a Account → API keys & tokens → Create API key. Se generan un API Key SID (empieza por SK) y un Secret. El Secret se muestra una sola vez: cópialo en ese momento. Las API keys se pueden revocar individualmente sin afectar el resto de la cuenta.

Verifica el flujo actual La consola de Twilio cambia con frecuencia. Confirma los nombres exactos de los menús contra la consola real al momento de configurar.

4. Permisos y credenciales

El conector solo necesita credenciales que permitan enviar mensajes y leer su estado. Para producción se recomienda una API Key sobre el Auth Token, por el principio de mínimo privilegio y revocación granular.

Credencial Para qué la usa el conector
Account SID Identifica la cuenta a la que pertenecen los mensajes (siempre requerido)
Auth Token o API Key SID + Secret Autentica las llamadas a la API (enviar y consultar)
Número remitente o Messaging Service SID Define desde qué remitente sale el SMS

5. Configuración en Ruvic

Entra a Settings → Conectores, busca la tarjeta Twilio SMS y pulsa Configurar. Elige el modo que preparaste en la sección 3 y diligencia sus campos. En ambos modos debes indicar además el remitente (número O Messaging Service):

Campo del formulario Valor (obtenido en la sección 3)
Account SID El Account SID (empieza por AC)
Auth Token (modo A) El Auth Token del panel. Se guarda como secreto
API Key SID + Secret (modo B) El SID (SK…) y el Secret de la API Key. El Secret se guarda como secreto
Número remitente Tu número de Twilio en E.164 (ej. +15017122661)
Messaging Service SID Alternativa al número (MG…). Si defines ambos, prevalece este
Timeout de peticiones Opcional; 15 segundos por defecto

Guarda y pulsa Probar conexión — la plataforma validará las credenciales consultando la cuenta, sin enviar ningún SMS.

[ Captura: formulario del conector Twilio SMS en Settings → Conectores — reemplazar por assets/img/twilio-form.png ]

6. Verificación

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa a la cuenta Twilio AC…”, indicando el modo y el remitente. La tarjeta del conector cambia a estado Configurado. Este test no envía SMS ni genera costo.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele (usa un número verificado si tu cuenta es trial):

    “Envía un SMS al +57… que diga: Conector Ruvic configurado ✅, y luego dime si fue entregado.”

    Debe llegar el SMS y el agente reportar el estado. Si ocurre, el conector quedó operativo de punta a punta.

7. Solución de problemas

Mensaje / síntoma Causa probable y solución
“Autenticación fallida (HTTP 401)” Account SID, Auth Token o API Key incorrectos. Revísalos en la consola y vuelve a guardar el formulario.
Código 21211 — número 'to' inválido El número destino no está en formato E.164. Debe empezar por + y el código de país (ej. +573001234567).
Código 21608 — número no verificado Cuenta trial enviando a un número no verificado. Verifícalo en Verified Caller IDs o actualiza la cuenta.
Código 21610 — destinatario dado de baja El destinatario respondió STOP y bloqueó los mensajes de ese remitente. No se le puede enviar hasta que reactive.
El SMS queda en queued o undelivered Falta verificación regulatoria (toll-free/10DLC) o el número destino no recibe SMS. Revisa el error_code con consultar estado y las verificaciones del remitente.
“Falta el remitente” al guardar No se indicó ni número ni Messaging Service. Completa uno de los dos campos.

8. Preguntas frecuentes y notas de seguridad

¿Auth Token o API Key?

Para pruebas, el Auth Token es lo más rápido. Para producción, Twilio recomienda una API Key: se puede revocar de forma individual sin rotar el Auth Token de toda la cuenta, lo que limita el impacto si se compromete.

¿El agente confirma que el SMS llegó?

La entrega es asíncrona: al enviar, el estado es queued/sending. Para confirmar la entrega, el agente consulta el estado unos segundos después (operación sms.get_status), que evoluciona a delivered, undelivered o failed.

¿Dónde quedan guardadas las credenciales?

El Auth Token y el API Key Secret se guardan como secretos en el almacén de credenciales de la plataforma. La interfaz nunca vuelve a mostrarlos; solo indica “definido”. El Account SID y el número remitente no son secretos.

Rotación de credenciales

API Key: crea una nueva en la consola, actualízala en el formulario y elimina la anterior. Auth Token: usa la rotación de Auth Token secundario de Twilio para no interrumpir el servicio, luego promueve el nuevo y actualízalo en el formulario.

¿Qué hago si una credencial se compromete?

  1. API Key: elimínala de inmediato en API keys & tokens (deja de funcionar al instante) y crea otra.
  2. Auth Token: rótalo desde la consola (afecta a toda la cuenta).
  3. Actualiza la nueva credencial en el formulario del conector y revisa el historial de mensajes por envíos no autorizados.
Buenas prácticas Usa API keys dedicadas para Ruvic; no compartas tokens ni secrets por canales no seguros ni los publiques en repositorios; envía solo a destinatarios que hayan dado consentimiento; y trata los códigos OTP como datos sensibles (no los registres en logs ni los repitas innecesariamente).
