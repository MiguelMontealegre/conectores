1. ¿Qué es y para qué sirve?1. What is it and what is it for?

El conector Twilio SMS permite que el agente de Ruvic envíe mensajes de texto (SMS) a cualquier número de teléfono y consulte si fueron entregados. Es ideal para mensajes transaccionales. Una vez configurado, puedes pedirle al agente cosas como:

  • “Envía un código de verificación por SMS al número del cliente.”
  • “Cuando el pedido se despache, notifícalo por SMS al comprador.”
  • “¿El SMS de alerta que enviamos hace un momento llegó?”

Operaciones soportadas:

The Twilio SMS connector lets the Ruvic agent send text messages (SMS) to any phone number and check whether they were delivered. It's ideal for transactional messages. Once configured, you can ask the agent things like:

  • “Send a verification code by SMS to the customer's number.”
  • “When the order ships, notify the buyer by SMS.”
  • “Did the alert SMS we just sent get delivered?”

Supported operations:

OperaciónOperation Qué haceWhat it does
sms.send Envía un SMS a un número en formato internacionalSends an SMS to a number in international format
sms.get_status Consulta el estado de entrega de un mensaje enviadoChecks the delivery status of a sent message
sms.list Lista los mensajes recientes de la cuenta (con filtros)Lists the account's recent messages (with filters)
Envío, no recepciónSending, not receiving Este conector es de solo salida: envía SMS y consulta su estado. Recibir SMS entrantes requeriría un endpoint público permanente, incompatible con el modelo de ejecución de Ruvic. This connector is outbound-only: it sends SMS and checks their status. Receiving inbound SMS would require a permanent public endpoint, incompatible with Ruvic's execution model.

2. Prerrequisitos2. Prerequisites

  • Una cuenta de Twilio (twilio.com/console). Sirve una cuenta trial para pruebas, con limitaciones (ver más abajo).
  • Un remitente: un número de teléfono de Twilio habilitado para SMS, o un Messaging Service.
  • Según el país destino, las verificaciones regulatorias correspondientes (verificación toll-free en EE. UU./Canadá, registro 10DLC, o bundle regulatorio en otros países).
  • Saldo en la cuenta: cada SMS enviado tiene costo (salvo credenciales de prueba).
  • A Twilio account (twilio.com/console). A trial account works for testing, with limitations (see below).
  • A sender: a Twilio phone number enabled for SMS, or a Messaging Service.
  • Depending on the destination country, the relevant regulatory verifications (toll-free verification in the US/Canada, 10DLC registration, or a regulatory bundle elsewhere).
  • Account balance: every SMS sent has a cost (except with test credentials).
Cuentas trialTrial accounts Una cuenta trial de Twilio solo puede enviar a números previamente verificados en la consola, y antepone un texto de prueba a cada mensaje. Para producción, actualiza la cuenta. A Twilio trial account can only send to numbers previously verified in the console, and prepends a trial notice to each message. For production, upgrade the account.

3. Configuración en Twilio (paso a paso)3. Twilio-side setup (step by step)

  1. Consigue un remitente.

    En la consola de Twilio, ve a Phone Numbers y compra (o usa) un número habilitado para SMS. Anótalo en formato E.164 (ej. +15017122661). Alternativamente, crea un Messaging Service en Messaging → Services y anota su SID (empieza por MG).

    Get a sender.

    In the Twilio console, go to Phone Numbers and buy (or use) a number enabled for SMS. Note it in E.164 format (e.g. +15017122661). Alternatively, create a Messaging Service under Messaging → Services and note its SID (starts with MG).

    [ Captura: número SMS o Messaging Service en la consola de Twilio — reemplazar por assets/img/twilio-sender.png ] [ Screenshot: SMS number or Messaging Service in the Twilio console — replace with assets/img/twilio-sender.png ]
  2. (Cuenta trial) Verifica el número destino.

    Si tu cuenta es trial, ve a Phone Numbers → Verified Caller IDs y verifica cada número al que quieras enviar SMS de prueba. Sin esto, el envío fallará con el código 21608.

    (Trial account) Verify the destination number.

    If your account is trial, go to Phone Numbers → Verified Caller IDs and verify each number you want to send test SMS to. Without this, sending fails with code 21608.

  3. Elige tus credenciales — Opción A: Account SID + Auth Token.

    En el panel principal de la consola verás el Account SID (empieza por AC) y el Auth Token. Cópialos. Es lo más rápido, ideal para pruebas.

    Choose your credentials — Option A: Account SID + Auth Token.

    On the console's main dashboard you'll see the Account SID (starts with AC) and the Auth Token. Copy them. This is the fastest path, ideal for testing.

    [ Captura: Account SID y Auth Token en el panel (token ofuscado) — reemplazar por assets/img/twilio-credentials.png ] [ Screenshot: Account SID and Auth Token on the dashboard (token obfuscated) — replace with assets/img/twilio-credentials.png ]
  4. Opción B (recomendada para producción): crea una API Key.

    Ve a Account → API keys & tokens → Create API key. Se generan un API Key SID (empieza por SK) y un Secret. El Secret se muestra una sola vez: cópialo en ese momento. Las API keys se pueden revocar individualmente sin afectar el resto de la cuenta.

    Option B (recommended for production): create an API Key.

    Go to Account → API keys & tokens → Create API key. This generates an API Key SID (starts with SK) and a Secret. The Secret is shown only once: copy it right then. API keys can be revoked individually without affecting the rest of the account.

Verifica el flujo actualVerify the current flow La consola de Twilio cambia con frecuencia. Confirma los nombres exactos de los menús contra la consola real al momento de configurar. The Twilio console changes often. Confirm the exact menu names against the live console when configuring.

4. Permisos y credenciales4. Permissions and credentials

El conector solo necesita credenciales que permitan enviar mensajes y leer su estado. Para producción se recomienda una API Key sobre el Auth Token, por el principio de mínimo privilegio y revocación granular.

The connector only needs credentials that allow sending messages and reading their status. For production, an API Key is recommended over the Auth Token, for least-privilege and granular revocation.

6. Verificación6. Verification

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa a la cuenta Twilio AC…”, indicando el modo y el remitente. La tarjeta del conector cambia a estado Configurado. Este test no envía SMS ni genera costo.

    Successful connection test.

    The Test connection button should return: “Successful connection to Twilio account AC…”, indicating the mode and sender. The connector card switches to Configured. This test sends no SMS and incurs no cost.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele (usa un número verificado si tu cuenta es trial):

    “Envía un SMS al +57… que diga: Conector Ruvic configurado ✅, y luego dime si fue entregado.”

    Debe llegar el SMS y el agente reportar el estado. Si ocurre, el conector quedó operativo de punta a punta.

    Minimal test automation.

    Open a new conversation with the agent and ask (use a verified number if your account is trial):

    “Send an SMS to +57… saying: Ruvic connector configured ✅, then tell me if it was delivered.”

    The SMS should arrive and the agent report the status. If so, the connector is fully operational.

8. Preguntas frecuentes y notas de seguridad8. FAQ and security notes

¿Auth Token o API Key?Auth Token or API Key?

Para pruebas, el Auth Token es lo más rápido. Para producción, Twilio recomienda una API Key: se puede revocar de forma individual sin rotar el Auth Token de toda la cuenta, lo que limita el impacto si se compromete.

For testing, the Auth Token is fastest. For production, Twilio recommends an API Key: it can be revoked individually without rotating the whole account's Auth Token, limiting the blast radius if compromised.

¿El agente confirma que el SMS llegó?Does the agent confirm the SMS arrived?

La entrega es asíncrona: al enviar, el estado es queued/sending. Para confirmar la entrega, el agente consulta el estado unos segundos después (operación sms.get_status), que evoluciona a delivered, undelivered o failed.

Delivery is asynchronous: on send, the status is queued/sending. To confirm delivery, the agent checks the status a few seconds later (sms.get_status), which evolves to delivered, undelivered or failed.

¿Dónde quedan guardadas las credenciales?Where are the credentials stored?

El Auth Token y el API Key Secret se guardan como secretos en el almacén de credenciales de la plataforma. La interfaz nunca vuelve a mostrarlos; solo indica “definido”. El Account SID y el número remitente no son secretos.

The Auth Token and API Key Secret are stored as secrets in the platform's credential store. The UI never displays them again; it only shows “set”. The Account SID and sender number are not secret.

Rotación de credencialesCredential rotation

API Key: crea una nueva en la consola, actualízala en el formulario y elimina la anterior. Auth Token: usa la rotación de Auth Token secundario de Twilio para no interrumpir el servicio, luego promueve el nuevo y actualízalo en el formulario.

API Key: create a new one in the console, update it in the form and delete the old one. Auth Token: use Twilio's secondary Auth Token rotation to avoid downtime, then promote the new one and update it in the form.

¿Qué hago si una credencial se compromete?What if a credential is compromised?

  1. API Key: elimínala de inmediato en API keys & tokens (deja de funcionar al instante) y crea otra.
  2. Auth Token: rótalo desde la consola (afecta a toda la cuenta).
  3. Actualiza la nueva credencial en el formulario del conector y revisa el historial de mensajes por envíos no autorizados.
  1. API Key: delete it immediately in API keys & tokens (stops working instantly) and create another.
  2. Auth Token: rotate it from the console (affects the whole account).
  3. Update the new credential in the connector form and review the message history for unauthorized sends.
Buenas prácticasBest practices Usa API keys dedicadas para Ruvic; no compartas tokens ni secrets por canales no seguros ni los publiques en repositorios; envía solo a destinatarios que hayan dado consentimiento; y trata los códigos OTP como datos sensibles (no los registres en logs ni los repitas innecesariamente). Use dedicated API keys for Ruvic; never share tokens or secrets over insecure channels or commit them to repositories; only message recipients who have opted in; and treat OTP codes as sensitive data (don't log them or repeat them unnecessarily).
