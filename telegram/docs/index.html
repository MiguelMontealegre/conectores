1. ¿Qué es y para qué sirve?1. What is it and what is it for?

El conector Telegram permite que el agente de Ruvic se comunique a través de un bot de Telegram: enviar mensajes y archivos a personas, grupos y canales, y leer los mensajes que llegan al bot. Una vez configurado, puedes pedirle al agente cosas como:

  • “Cuando termine el reporte de ventas, envíalo al grupo de Operaciones por Telegram.”
  • “Avísame por Telegram si algún servidor supera el 90% de disco.”
  • “Revisa los mensajes que llegaron al bot y resúmeme las solicitudes.”

Operaciones soportadas:

The Telegram connector lets the Ruvic agent communicate through a Telegram bot: send messages and files to people, groups and channels, and read the messages the bot receives. Once configured, you can ask the agent things like:

  • “When the sales report is ready, send it to the Operations group on Telegram.”
  • “Notify me on Telegram if any server goes above 90% disk usage.”
  • “Check the messages the bot received and summarize the requests.”

Supported operations:

OperaciónOperation Qué haceWhat it does
chat.send_message Envía un mensaje de texto a un chat, grupo o canalSends a text message to a chat, group or channel
chat.send_file Envía un archivo (documento, foto, video o audio)Sends a file (document, photo, video or audio)
chat.get_updates Recibe los mensajes y actualizaciones entrantes del botReceives the bot's incoming messages and updates
chat.get_info Consulta la información de un chat, grupo o canalGets information about a chat, group or channel
Todo pasa por un botEverything goes through a bot Ruvic no usa tu cuenta personal de Telegram: actúa mediante un bot dedicado que tú creas. El bot solo ve los chats a los que lo agregas y los mensajes que le envían directamente. Ruvic does not use your personal Telegram account: it acts through a dedicated bot you create. The bot only sees the chats you add it to and the messages sent directly to it.

2. Prerrequisitos2. Prerequisites

  • Una cuenta de Telegram (la app en tu teléfono o Telegram Desktop) para hablar con BotFather.
  • El chat, grupo o canal destino donde el bot enviará mensajes, y permiso para agregar el bot ahí (ser administrador del grupo/canal, o iniciar tú el chat privado con el bot).
  • Nada de infraestructura extra: no se necesita servidor propio; Ruvic se comunica con la API pública de Telegram (api.telegram.org).
  • A Telegram account (the phone app or Telegram Desktop) to talk to BotFather.
  • The target chat, group or channel where the bot will send messages, and permission to add the bot there (be a group/channel admin, or start the private chat with the bot yourself).
  • No extra infrastructure: no server needed; Ruvic talks to Telegram's public API (api.telegram.org).

3. Configuración en Telegram (paso a paso)3. Telegram-side setup (step by step)

Vas a crear un bot dedicado con BotFather (el bot oficial de Telegram para gestionar bots), obtener su token y el ID del chat destino. Todo se hace dentro de la app de Telegram.

You will create a dedicated bot with BotFather (Telegram's official bot for managing bots), get its token and the target chat ID. Everything is done inside the Telegram app.

  1. Crea el bot con BotFather.

    En Telegram, busca @BotFather, ábrelo y envía /newbot. Te pedirá un nombre visible y un username único que debe terminar en bot (ej. ruvic_ops_bot).

    Create the bot with BotFather.

    In Telegram, search for @BotFather, open it and send /newbot. It will ask for a display name and a unique username that must end in bot (e.g. ruvic_ops_bot).

    [ Captura: conversación con BotFather ejecutando /newbot — reemplazar por assets/img/telegram-newbot.png ] [ Screenshot: BotFather conversation running /newbot — replace with assets/img/telegram-newbot.png ]
  2. Copia el token del bot.

    Al terminar, BotFather te entrega un token con el formato 123456789:AAH8x.... Guárdalo: es la credencial que irá en el formulario de Ruvic (campo Token del bot). Trátalo como una contraseña — quien lo tenga controla el bot.

    Copy the bot token.

    When done, BotFather gives you a token in the form 123456789:AAH8x.... Save it: it is the credential that goes in the Ruvic form (Bot token field). Treat it like a password — anyone who has it controls the bot.

    123456789:AAH8xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  3. Agrega el bot al destino.
    • Chat privado: abre el chat con tu bot y pulsa Iniciar (los bots no pueden escribirle a alguien que nunca les habló).
    • Grupo: agrega el bot como miembro del grupo.
    • Canal: agrega el bot como administrador con permiso de publicar.
    Add the bot to the target.
    • Private chat: open the chat with your bot and tap Start (bots can't message someone who never wrote to them first).
    • Group: add the bot as a member of the group.
    • Channel: add the bot as an administrator with permission to post.
  4. Obtén el ID del chat destino.

    Escribe cualquier mensaje en el chat/grupo donde está el bot. Luego abre en el navegador (reemplaza <TOKEN> por tu token):

    Get the target chat ID.

    Send any message in the chat/group where the bot is. Then open in your browser (replace <TOKEN> with your token):

    https://api.telegram.org/bot<TOKEN>/getUpdates

    En la respuesta JSON, busca "chat":{"id": ...}. Ese número es el ID: positivo para chats privados, y negativo empezando por -100 para supergrupos y canales. Lo usarás en el campo Chat por defecto.

    In the JSON response, find "chat":{"id": ...}. That number is the ID: positive for private chats, negative starting with -100 for supergroups and channels. You'll use it in the Default chat field.

    [ Captura: respuesta de getUpdates con el chat.id resaltado (datos ofuscados) — reemplazar por assets/img/telegram-chatid.png ] [ Screenshot: getUpdates response with chat.id highlighted (data obfuscated) — replace with assets/img/telegram-chatid.png ]
  5. (Opcional) Permite leer todos los mensajes de un grupo.

    Por defecto, en grupos el bot solo ve comandos y menciones (privacy mode). Si necesitas que get_updates lea todos los mensajes, en BotFather envía /setprivacy, elige tu bot y selecciona Disable. Luego quita y vuelve a agregar el bot al grupo para que aplique.

    (Optional) Allow reading all group messages.

    By default, in groups the bot only sees commands and mentions (privacy mode). If you need get_updates to read all messages, in BotFather send /setprivacy, choose your bot and select Disable. Then remove and re-add the bot to the group for it to take effect.

4. Permisos requeridos4. Required permissions

Principio de mínimo privilegio: el bot solo necesita estar presente en el destino y, según el caso, poder publicar o leer. No requiere permisos de administración salvo para publicar en canales.

Least-privilege principle: the bot only needs to be present in the target and, depending on the case, be able to post or read. No admin permissions required except to post in channels.

Administrador con permiso de publicar (solo canales)Administrator with post permission (channels only) Publicar mensajes en un canal (los canales no admiten miembros normales que publiquen)Post messages to a channel (channels don't allow regular members to post)
Privacy mode desactivado (opcional, grupos)Privacy mode disabled (optional, groups) Leer todos los mensajes del grupo, no solo comandos y mencionesRead all group messages, not just commands and mentions

Entra a Settings → Conectores, busca la tarjeta Telegram y pulsa Configurar. Diligencia el formulario con los valores que obtuviste en la sección 3:

Go to Settings → Connectors, find the Telegram card and click Configure. Fill in the form with the values from section 3:

6. Verificación6. Verification

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa como @tu_bot” (y, si configuraste un chat por defecto, mencionará el acceso a ese chat). La tarjeta del conector cambia a estado Configurado.

    Successful connection test.

    The Test connection button should return: “Successful connection as @your_bot” (and, if you set a default chat, it will mention access to that chat). The connector card switches to Configured.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele:

    “Envía un mensaje de prueba por Telegram que diga: Conector configurado ✅.”

    El mensaje debe aparecer en el chat destino. Si llega, el conector quedó operativo de punta a punta.

    Minimal test automation.

    Open a new conversation with the agent and ask:

    “Send a test message on Telegram saying: Connector configured ✅.”

    The message should appear in the target chat. If it arrives, the connector is fully operational.

8. Preguntas frecuentes y notas de seguridad8. FAQ and security notes

¿Ruvic accede a mi cuenta personal de Telegram?Does Ruvic access my personal Telegram account?

No. Todo ocurre a través del bot que creaste. El bot solo ve los chats a los que lo agregas y los mensajes que le envían; nunca accede a tus chats personales.

No. Everything happens through the bot you created. The bot only sees the chats you add it to and the messages sent to it; it never accesses your personal chats.

¿Dónde queda guardado el token?Where is the token stored?

Como secreto en el almacén de credenciales de la plataforma, asociado a tu usuario. La interfaz nunca vuelve a mostrarlo; solo indica “definido”. Para cambiarlo, escribe uno nuevo en el formulario; para conservarlo al editar otros campos, deja el campo vacío.

As a secret in the platform's credential store, tied to your user. The UI never displays it again; it only shows “set”. To change it, type a new one in the form; to keep it while editing other fields, leave the field empty.

Rotación de credencialesCredential rotation

Para rotar el token, envía /token a BotFather, elige tu bot y genera uno nuevo; actualízalo en el formulario del conector y pulsa Probar conexión. El token anterior deja de funcionar al instante.

To rotate the token, send /token to BotFather, pick your bot and generate a new one; update it in the connector form and click Test connection. The old token stops working immediately.

¿Qué hago si el token se compromete?What if the token is compromised?

  1. En BotFather, envía /revoke, elige el bot y revoca el token de inmediato (invalida el actual y genera uno nuevo).
  2. Actualiza el nuevo token en el formulario del conector.
  3. Revisa en el chat/grupo si hubo mensajes no autorizados enviados por el bot.
  1. In BotFather, send /revoke, pick the bot and revoke the token immediately (invalidates the current one and issues a new one).
  2. Update the new token in the connector form.
  3. Check the chat/group for any unauthorized messages sent by the bot.

¿Expira el token?Does the token expire?

No caduca por sí solo: sigue válido hasta que lo revoques en BotFather. Aun así, se recomienda rotarlo periódicamente según tu política de seguridad.

It does not expire on its own: it stays valid until you revoke it in BotFather. Still, rotating it periodically per your security policy is recommended.

Buenas prácticasBest practices Usa un bot dedicado solo para Ruvic; no compartas el token por canales no seguros ni lo publiques en repositorios; agrega el bot únicamente a los chats estrictamente necesarios; y mantén el privacy mode activado salvo que realmente necesites leer todos los mensajes de un grupo. Use a dedicated bot only for Ruvic; never share the token over insecure channels or commit it to repositories; add the bot only to the chats strictly needed; and keep privacy mode enabled unless you truly need to read all group messages.
