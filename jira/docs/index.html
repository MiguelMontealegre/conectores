Conector Jira

Última actualización: 21 de julio de 2026

1. ¿Qué es y para qué sirve?

El conector Jira permite que el agente de Ruvic gestione tickets en Jira Cloud: crear incidencias, actualizar sus campos, cambiar su estado, comentar y buscar con JQL. Una vez configurado, puedes pedirle al agente cosas como:

  • “Crea un bug en el proyecto OPS: el nodo 3 no responde, prioridad alta.”
  • “Mueve el ticket OPS-123 a En progreso y comenta que ya empecé.”
  • “Lístame los tickets abiertos de OPS creados esta semana.”

Operaciones soportadas:

Operación Qué hace
issue.create Crea un ticket (incidencia) en un proyecto
issue.update Actualiza campos: resumen, descripción, prioridad, etiquetas, asignado
issue.transition Cambia el estado del ticket según el flujo de trabajo
issue.add_comment Añade un comentario a un ticket
issue.search Busca tickets con una consulta JQL
issue.get Obtiene un ticket por su clave
Actúa con una cuenta de Atlassian Ruvic opera Jira mediante un API token de una cuenta de Atlassian que tú designes. Todas las acciones (crear, comentar, transicionar) usan los permisos de esa cuenta y quedan registradas a su nombre.

2. Prerrequisitos

  • Un sitio de Jira Cloud (https://tuempresa.atlassian.net).
  • Una cuenta de Atlassian con permisos en el proyecto donde se gestionarán los tickets (crear, editar, transicionar, comentar).
  • La clave del proyecto destino (el prefijo de los tickets, ej. OPS en OPS-123).
Solo Jira Cloud Este conector usa autenticación de Jira Cloud (email + API token). Jira Server / Data Center usa otro mecanismo (Personal Access Tokens) y no está cubierto aquí.

3. Configuración en Atlassian (paso a paso)

Vas a generar un API token para tu cuenta de Atlassian y a anotar la URL del sitio y la clave del proyecto. No hace falta crear ninguna app.

  1. Anota la URL de tu sitio de Jira.

    Es la dirección con la que entras a Jira, del tipo https://tuempresa.atlassian.net. La usarás en el campo URL del sitio.

  2. Crea un API token.

    Entra a id.atlassian.com/manage-profile/security/api-tokens, pulsa Create API token, ponle un nombre (ej. Ruvic) y copia el token. Solo se muestra una vez; guárdalo como una contraseña.

    ATATT3xFfGF0xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    [ Captura: pantalla Create API token en id.atlassian.com (token ofuscado) — reemplazar por assets/img/jira-token.png ]
  3. Identifica el email de la cuenta.

    Es el correo con el que inicias sesión en Atlassian. Va junto al token: Jira Cloud autentica con la pareja email + API token.

  4. Anota la clave del proyecto.

    Abre el proyecto en Jira: la clave es el prefijo en mayúsculas de sus tickets (ej. OPS). También aparece en la URL de cualquier ticket (/browse/OPS-123). La usarás como proyecto por defecto (opcional).

  5. Verifica los permisos de la cuenta.

    La cuenta debe poder, en ese proyecto: ver (Browse projects), crear, editar, transicionar y comentar incidencias. Si no puede, pide a un administrador de Jira que le otorgue esos permisos.

4. Permisos requeridos

Principio de mínimo privilegio: usa una cuenta con permiso solo en los proyectos necesarios. El conector nunca hace más de lo que la cuenta puede hacer manualmente.

Permiso de proyecto Para qué lo usa el conector
Browse projects Buscar y leer tickets (search, get)
Create issues Crear tickets (create)
Edit issues Actualizar campos (update)
Transition issues Cambiar el estado (transition)
Add comments Comentar tickets (add_comment)

5. Configuración en Ruvic

Entra a Settings → Conectores, busca la tarjeta Jira y pulsa Configurar. Diligencia el formulario con los valores de la sección 3:

Campo del formulario Valor (obtenido en la sección 3)
URL del sitio de Jira Ej. https://tuempresa.atlassian.net
Email de la cuenta Atlassian El correo con el que inicias sesión en Atlassian
API token El token del paso 2. Se guarda cifrado como secreto: la interfaz no volverá a mostrarlo
Proyecto por defecto Opcional. La clave del proyecto (ej. OPS) para crear/buscar cuando no se indique otro
Timeout de peticiones Opcional; 20 segundos por defecto

Guarda y pulsa Probar conexión — la plataforma validará las credenciales y, si configuraste un proyecto por defecto, verificará que la cuenta pueda acceder a él.

6. Verificación

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa como Tu Nombre” (y, si configuraste un proyecto por defecto, mencionará el acceso a ese proyecto). La tarjeta cambia a estado Configurado.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele:

    “Crea un ticket de prueba en el proyecto OPS con el resumen: Prueba de conector.”

    El ticket debe aparecer en Jira con su clave (ej. OPS-124). Si se crea, el conector quedó operativo de punta a punta. Puedes borrarlo después.

7. Solución de problemas

Mensaje / síntoma Causa probable y solución
“Autenticación o permisos insuficientes (HTTP 401)” Email o API token incorrectos. Verifica el email exacto de la cuenta y regenera el token en id.atlassian.com si hace falta.
“…insuficientes (HTTP 403)” La cuenta no tiene permiso para esa acción en el proyecto. Pide a un administrador que le conceda Create/Edit/Transition/Add comments.
“Recurso no encontrado” La clave del ticket o del proyecto no existe o no es visible para la cuenta. Verifica la clave y los permisos de visualización.
“No hay una transición hacia ese estado” El flujo de trabajo no permite pasar a ese estado desde el actual. El mensaje lista los estados alcanzables; elige uno de ellos.
Error al buscar con JQL La consulta JQL tiene errores de sintaxis o nombres de campo inválidos. Revisa comillas, operadores y nombres de estado (entre comillas si tienen espacios).
“issuetype is required / invalid” El tipo de issue no existe en ese proyecto. Usa un tipo válido del proyecto (Task, Bug, Story…) o consulta con un administrador.

8. Preguntas frecuentes y notas de seguridad

¿Con qué usuario se crean los tickets?

Con la cuenta de Atlassian dueña del API token. Los tickets y comentarios aparecen a su nombre, y solo puede hacer lo que sus permisos le permitan.

¿Dónde queda guardado el API token?

Como secreto en el almacén de credenciales de la plataforma, asociado a tu usuario. La interfaz nunca vuelve a mostrarlo; solo indica “definido”. Para cambiarlo, escribe uno nuevo; para conservarlo al editar otros campos, deja el campo vacío.

Rotación de credenciales

Crea un token nuevo en id.atlassian.com, actualízalo en el formulario del conector y pulsa Probar conexión; luego revoca el token anterior desde la misma página de Atlassian.

¿Qué hago si el token se compromete?

  1. En id.atlassian.com → API tokens, revoca el token comprometido de inmediato.
  2. Crea uno nuevo y actualízalo en el formulario del conector.
  3. Revisa el historial de actividad de la cuenta en Jira por cambios no autorizados.
Buenas prácticas Usa una cuenta de servicio dedicada con permisos solo en los proyectos necesarios; no compartas el token por canales inseguros ni lo publiques en repositorios; y rota el token periódicamente según tu política de seguridad.
